# 03 页面模块与组件设计

## 现有架构保留

```text
App
└─ CourseShell
   ├─ CourseProgress
   ├─ CourseModuleView
   ├─ TeacherScriptPanel
   └─ CourseTimeline
```

## v2 页面模块

`CourseModuleView` 继续作为模块分发器，新增或增强以下视图：

- 课程封面：暴雨山区快速导入
- 定义判断流程
- 六类灾害总览
- 形成原因实验室
- 四川重点三类入口
- 滑坡完整演示
- 崩塌完整演示
- 泥石流完整演示
- 科学避险流程 + 前兆分类墙
- 另外三类拓展 + 总结口诀

## 演示复用策略

- 滑坡继续使用 `LandslideDemo`。
- 崩塌和泥石流新增轻量演示组件，先满足课堂展示和录屏。
- 不强行抽象复杂 Demo 引擎，避免当前阶段过度设计。

## 数据拆分

- `courseModules.ts`：课程主线和教师讲稿。
- `disasterCards.ts`：六类灾害卡片。
- `safetyRules.ts`：避险流程、前兆分类、口诀。

## 组件边界

- 课程层可以管理模块切换、滑杆值、轻量点击反馈。
- Presentation Core 只管理灾害演示阶段播放状态。
- 灾害 Scene 组件只负责展示。
